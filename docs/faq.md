## 关于数值本身

### Q1：这套数值来自真实的商业项目吗？
**A**：是的。这套数值体系直接来自市场上运营超过 12 个月的头部捕鱼游戏（鱼机类型），日活用户超过 10 万。所有参数都经过真实玩家数据验证，不是理论设计。

### Q2：这套数值适合什么样的捕鱼游戏？
**A**：主要适合**炮台射击类捕鱼游戏**（即玩家控制炮台射击鱼群），包括：
- 街机风格（Arcade）
- 休闲手机游戏（Casual Mobile）
- 中度混合变现游戏（Hybrid-Casual）

不太适合纯放置类或文字类捕鱼游戏。

### Q3：什么是“数值文案”？它和普通文案有什么区别？
**A**：
| 类型 | 示例 | 作用 |
| :--- | :--- | :--- |
| 普通文案 | “恭喜你获得大奖！” | 提升情绪 |
| 数值文案 | 鱼的 HP=20, 炮伤=10, 暴击率=8% | 决定游戏规则和体验 |

数值文案是游戏的“骨架”，决定了玩家觉得好玩还是无聊。

### Q4：不同语言版本（简中/繁中/英文）的数值一样吗？
**A**：**数值完全一样**，只是参数名的注释和文档提供了三种语言。因为数值配置与语言无关，只与游戏设计逻辑有关。

### Q5：你们有提供美术资源包吗？
**A**：本仓库专注于**数值系统**。如果你需要配套的美术资源（鱼 sprite、炮台特效、UI 素材），可以通过 Telegram 联系我们，我们可以提供合作的第三方资源包信息。

## 关于使用和集成

### Q6：数值文件是什么格式？我能直接导入 Unity 或 Cocos 吗？
**A**：提供两种格式：
- **Excel (.xlsx)**：方便你在表格中查看和修改
- **JSON (.json)**：可被大多数游戏引擎直接解析

导入示例（Unity C#）：

string json = File.ReadAllText(Application.dataPath + "/fish_config.json");
FishConfig config = JsonUtility.FromJson<FishConfig>(json);

### Q7：我可以只使用其中一部分数值吗？



A：可以。模块是解耦的，例如：

只想要鱼类 HP/分值 → 只复制 fish_config/

只想要 Boss 机制 → 只复制 boss_config/

建议保留参数命名格式，以免后续升级时冲突。

### Q8：我的游戏已经上线了，能替换成这套数值吗？
A：可以，但建议分两步：

先在测试服完整替换，运行 3~5 天观察经济曲线

正式服对 10% 玩家开启 A/B 测试（使用 reward_config 中的分流配置）

确认核心指标（时长、留存）正向后再全量发布

### Q9：这套数值在 HTML5 / Web 游戏上性能如何？
A：数值计算本身极轻量（纯配置表，无复杂实时计算）。即使在 H5 环境，每秒处理 1000 次命中判定也毫无压力。性能瓶颈在于渲染和碰撞检测，与数值系统无关。
